## MODIFIED Requirements

### Requirement: 右侧边栏交互

系统 SHALL 提供一个右侧可展开的边栏，用于节点的创建和编辑。

#### Scenario: 点击创建节点按钮展开边栏

- **WHEN** 用户点击"创建节点"按钮
- **THEN** 右侧边栏展开，显示简化的节点编辑表单（仅包含内容字段）
- **AND** 画布区域保持可操作状态，无遮罩层阻挡

#### Scenario: 在边栏中创建节点

- **WHEN** 用户在边栏中填写节点内容并点击保存按钮
- **THEN** 在画布上创建新节点并显示内容
- **AND** 边栏自动收起

#### Scenario: 点击节点查看内容

- **WHEN** 用户点击画布上的已有节点
- **THEN** 右侧边栏展开，显示该节点的内容
- **AND** 用户可以编辑节点内容并通过保存按钮确认修改

#### Scenario: 拖拽连接线创建节点

- **WHEN** 用户从一个节点拖拽连接线到空白区域
- **THEN** 右侧边栏展开，显示简化的新节点创建表单
- **AND** 保存后创建目标节点并建立连接

### Requirement: 边栏布局和交互

边栏 MUST 提供简洁的用户界面和灵活的交互方式。

#### Scenario: 简化的表单设计

- **WHEN** 用户打开创建或编辑节点边栏
- **THEN** 界面只显示内容文本框和一个保存按钮
- **AND** 移除标题字段和取消按钮以简化操作

#### Scenario: 无遮罩层交互

- **WHEN** 边栏处于打开状态
- **THEN** 画布区域完全可见和可操作
- **AND** 用户可以在不关闭边栏的情况下操作画布

#### Scenario: 边栏关闭机制

- **WHEN** 用户需要关闭边栏
- **THEN** 可以通过点击关闭按钮或按ESC键关闭
- **AND** 点击画布区域不会关闭边栏
