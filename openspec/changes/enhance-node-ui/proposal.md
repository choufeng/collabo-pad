# enhance-node-ui 变更提案

## 概述

完善节点(node)的样式和交互功能，包括移除"+按钮"、修改层级显示为用户名，以及在侧边栏添加子节点创建功能。

## Why

现有的节点UI存在以下用户体验问题：

1. **冗余的+按钮**：节点上的+按钮功能与侧边栏编辑功能重复，造成界面混乱
2. **无意义的层级显示**：L0/L1等层级标识对最终用户没有实际价值，应该显示更有意义的信息
3. **缺失的快速创建功能**：在编辑父节点时，用户希望能够方便地创建子节点，而不需要多个步骤的操作

这些改进将提升界面的简洁性，提供更有意义的节点信息，并优化用户的创建工作流程。

## What Changes

- 移除节点上的"+"按钮，简化节点UI
- 将开发模式下的层级显示(L0/L1)改为创建者用户名显示
- 在侧边栏的Topic content表单下方添加分割线
- 在分割线下方添加子节点创建表单
- 实现自动创建父子节点关联的功能

## 变更内容

### 1. 移除节点"+"按钮

- 移除CustomNode组件中的"+"添加按钮
- 移除相关的onAddChild回调处理

### 2. 修改层级显示为用户名

- 将开发模式下的"L0"层级显示改为节点创建者的用户名
- 需要扩展节点数据结构以包含创建者信息

### 3. 侧边栏增强功能

- 在NodeEditor的Topic content表单下方添加分割线
- 添加子节点创建表单，用于创建与父节点关联的子topic
- 实现自动创建父子节点关联的功能

## 技术实现

- 修改CustomNode.tsx组件UI结构
- 扩展NodeData接口添加creator字段
- 增强NodeEditor.tsx组件功能
- 更新相关的数据管理逻辑

## 影响范围

- CustomNode组件
- NodeEditor组件
- RightSidebar组件
- 节点数据结构
- 用户数据服务
