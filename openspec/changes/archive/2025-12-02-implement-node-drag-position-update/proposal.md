## Why

为了提升用户体验，需要实现节点拖动位置的实时保存功能。当前用户可以拖动节点，但位置变化不会持久化到数据库，重新加载页面后节点位置会重置。

## What Changes

- 在Board组件中添加节点拖动结束事件处理
- 实现拖动位置变化检测和API调用
- 利用现有的 `/api/topics/update` 接口保存新的x,y坐标
- 添加适当的错误处理和用户反馈
- 优化性能，避免频繁的API调用

**影响范围：**

- 修改 `specs/board` 规范，增加拖动位置保存要求
- 影响 `src/components/Board.tsx` 添加拖动事件处理
- 可能影响 `src/components/CustomNode.tsx` 节点拖动行为
- 重用现有的 `src/app/api/topics/update` API

## Impact

- **用户体验提升：** 节点位置变化会自动保存，刷新页面后保持布局
- **交互优化：** 拖动操作更加自然，无需手动保存位置
- **数据一致性：** 前端显示与后端数据保持同步
- **性能考虑：** 需要合理控制API调用频率，避免过度请求
